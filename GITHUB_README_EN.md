## 🚀 Quick Start

### Three Steps to Complete Incident Investigation

```
Step 1                      Step 2                       Step 3
┌──────────────┐           ┌──────────────┐           ┌───────────────┐
│ Target Host  │   JSON    │ Analysis Host│           │  View Data    │
│ Run Agent    │ ───────>  │ Run GUI      │ ───────>  │  Search/Filter│
│ agent.exe    │(encrypted)│ getfact-gui  │           │  Hunt Threats │
└──────────────┘           └──────────────┘           └───────────────┘
```

**Run Agent on the target host:**
```bash
# Windows (run as Administrator recommended)
agent.exe

# Linux / Kylin
sudo ./agent_linux_amd64

# macOS
sudo ./agent_darwin_arm64
```

**Run GUI on the analysis host:**
```bash
getfact-gui.exe
# Click "Open File" → Load the encrypted report generated by Agent → Start analysis
```

## 📂 Output Details

### File Location

| Platform | Output Directory | Filename Format |
|----------|-----------------|-----------------|
| Windows | `C:\Getfact\` | `{hostname}_{timestamp}.json` |
| Linux/macOS | `/tmp/Getfact/` | `{hostname}_{timestamp}.json` |

### File Format

Agent output `.json` files are encrypted with **AES-256-GCM**. Opening them directly shows binary data. Use `getfact-gui.exe` to automatically decrypt and visualize.

```
Before Encryption (JSON)         After Encryption (Binary)
┌──────────────────────┐        ┌──────────────────────┐
│ {                    │  AES   │ ¥Þ¶×ŒÑ...           │
│   "version": "1.1",  │ ────>  │ (not human-readable) │
│   "modules": {...}   │  GCM   │                      │
│ }                    │        │                      │
└──────────────────────┘        └──────────────────────┘
```

## 🔐 Permissions

Some modules require **Administrator/root privileges** for full data collection:

| Module | Standard User | Administrator/root |
|--------|--------------|-------------------|
| Process List | Basic info | + Full command line + other users' processes |
| Network Connections | Connection info | + Associated process PID |
| Firewall | ✅ (Windows) | ✅ (Linux/macOS requires root) |
| Prefetch | ❌ No access | ✅ (Windows only) |
| System Logs | ✅ (Windows) | ✅ (Linux/macOS requires root) |
| GPU Info | ✅ Basic info | ✅ (nvidia-smi must be in PATH) |

**It is recommended to run Agent as Administrator/root for the most complete data collection.**

## ❓ FAQ

<details>
<summary><b>Q: Does the Agent require internet access?</b></summary>

No. The Agent runs entirely offline — it does not send any network requests or connect to external servers.
</details>

<details>
<summary><b>Q: The collected JSON file shows garbled text when opened directly?</b></summary>

This is expected. Since v1.1.0, Agent reports are encrypted with AES-256-GCM. Use `getfact-gui.exe` to open the file — the GUI will automatically decrypt it.
</details>

<details>
<summary><b>Q: Prefetch module shows no data?</b></summary>

Prefetch is only available on Windows and requires Administrator privileges. Right-click Agent → Run as Administrator.
</details>

<details>
<summary><b>Q: Network connection PID shows 0 on Linux?</b></summary>

Root privileges are required. Run with `sudo ./agent_linux_amd64`.
</details>

<details>
<summary><b>Q: GPU info only shows model, no utilization data?</b></summary>

Full GPU data (utilization, VRAM usage) requires NVIDIA drivers with `nvidia-smi` installed. Non-NVIDIA GPUs only provide basic model information.
</details>

<details>
<summary><b>Q: How to analyze data from Linux/macOS hosts?</b></summary>

Run Agent on the target host → Transfer the generated `.json` file to a Windows machine → Open and analyze with the GUI.
</details>
